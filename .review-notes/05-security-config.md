## Finding

I ran a direct source pass (not the diff-based native `/security-review`, since git is clean and the whole codebase predates this audit), reading the secret-handling, shell-out, path-confinement, and zip-extraction code end to end. Calibrated for a **local-first, single-user desktop tool** serving the operator's own legal case from `localhost`, the security posture is genuinely good: there are no hardcoded secrets, the one secret (`ANTHROPIC_API_KEY`) is read only from the environment / `.env` and never logged or printed, `.env` is gitignored with only a placeholder `.env.example` committed, and the two highest-risk surfaces — shelling out to the Python CLI and serving raw case files over HTTP — are mostly handled correctly. The biggest design-level mitigation is that the Bun `Bun.$` shell-outs use **tagged-template interpolation** everywhere (never string concatenation), so user-supplied values are auto-escaped as single argv tokens and command injection is not reachable through them.

The real defects are in **path confinement**, and they form a clean two-tier story. The download module (`web/download.ts`) does this *right*: a well-built `safeJoin` rejects null bytes, does a lexical `..`/absolute-path check, then `realpathSync`-resolves symlinks and re-confirms the result stays under a `realpath`'d root, and every byte-serving path (single PNG/PDF, and every staged bundle member) flows through it. By contrast the **static file handlers in `web/index.ts` do NOT use `safeJoin`** — `serveFromIntake` (`:50–55`, backing `/files/*` and `/pages/*`) and the inline `/png/*` handler (`:140–143`) guard only with a substring `decoded.includes("..")` check and then `join()` the user path directly. That string check is weaker than `safeJoin` in two concrete ways: (1) it does **no symlink resolution**, so a symlink planted inside `intake/<stem>/files/`, `intake/<stem>/pages/`, or `extracted/pages/` that points outside the root would be followed and its target served; and (2) it does not reject an absolute-path component, relying on `join()` semantics rather than an explicit deny. For this threat model the severity is **Medium-Low** — exploitation requires an attacker-controlled symlink already inside the case dir (i.e. a malicious intake zip whose extraction planted one, or a hostile filesystem) and the server binds to localhost — but it is a real inconsistency: two code paths confine the same roots, one robustly and one not, and the robust helper already exists three files away.

Two configuration notes round it out. First, `Bun.serve` binds the **default host** (`web/index.ts:57–58` sets only `port`), which under Bun means `0.0.0.0` — the viewer is reachable from any host on the LAN/VPN, and there is **no auth and no CORS restriction**, while it serves sensitive legal PDFs and full transcriptions. For a tool documented as "local, single-user" this is an insecure default worth a one-line fix (`hostname: "127.0.0.1"`). Second, zip-slip on intake is handled correctly: `extract_zip` (`reader.py:55–59`) pre-scans every entry with `is_suspicious_entry` (`models.py:80–84`, rejects absolute paths and any `..` component, normalizing `\` → `/`) and raises before any `extractall`. There is no automated dependency CVE audit; floors are pinned low (`anthropic>=0.69`, `duckdb>=1.1`, `pillow>=10.0`, `pypdfium2>=4.30`) with no upper bound, and `pillow`/`pypdfium2` parse untrusted court PDFs/images — worth a periodic `pip-audit`/Dependabot, but no known-exploited CVE was identified here.

## Evidence

- **Secrets — no hardcoded keys [PASS].** `grep -rniE "sk-ant|api[_-]?key|secret|password|token" src web` returns only doc strings, `os.environ.get("ANTHROPIC_API_KEY")` reads, and unrelated tokenizer/`token`-offset code (`regions.py`, `export.py`). No literal key, no `sk-ant-…` outside the placeholder in `.env.example`.
- **`.env` gitignored, no secret committed [PASS].** `.gitignore:18–20` ignores `.env`/`.env.local`; `git ls-files` shows only `.env.example` tracked. `.env.example` carries commented placeholders (`# ANTHROPIC_API_KEY=sk-ant-...`) — no real value.
- **API key handling [PASS].** Key read only from env: `transcribe.py:90` (SDK default client reads `ANTHROPIC_API_KEY`), `cli.py:261/508/553`. Never printed/logged (confirmed by Agent 04; no `logging` in `src/`, Rich console never emits it). `.env` sourced via `cli.py:311–318` `_load_dotenv()` with `override=False` (real exported env wins). **Severity: informational.**
- **Command injection via `Bun.$` [PASS — all interpolation].** Every shell-out uses tagged-template interpolation, not concatenation, so Bun auto-escapes each value as one argv token:
  - `index.ts:64` `export-actual --case-dir ${CASE_DIR}` (env-sourced)
  - `index.ts:74` `export-exploded --case-dir ${CASE_DIR}`
  - `index.ts:87` `export-metrics --case-dir ${CASE_DIR}`
  - `index.ts:109` `export-regions --case-dir ${CASE_DIR} --page ${pageId}` — `pageId` from `?page=` query string (`:107`), interpolated (safe; reaches argparse as a single arg).
  - `download.ts:110` `export-exploded --case-dir ${cfg.caseDir} ${args}` (`args` is a `string[]` — Bun escapes each element).
  - `download.ts:256/258/266` `mkdir -p ${dirname(dest)}` / `cp ${m.src} ${dest}` / `cd ${work} && zip -r -X -q ${archive} . -x __bundle.zip` — all interpolated; `dest`/`src` are `safeJoin`-confined paths, `work` is `mkdtemp`. No unescaped user string anywhere. **Severity: Low (residual).**
- **Path traversal — `safeJoin` is correct [PASS].** `download.ts:79–97`: rejects `\0` (`:81`), `realpathSync(resolve(root))` to canonicalize root (`:82`), lexical confinement `candidate.startsWith(realRoot + "/")` catches `..`/absolute even for missing files (`:85–87`), then symlink confinement via `realpathSync(candidate)` re-checked against root (`:89–92`). Used by `pngResponse` (`:157`), `pdfResponse` (`:170`), `pageMembers`/`imageMembers` (`:208,227`), and bundle staging (`:254`).
- **Path traversal — static handlers BYPASS `safeJoin` [Medium-Low].** `serveFromIntake` (`index.ts:50–55`, serves `/files/*` and `/pages/*`) and `/png/*` (`index.ts:140–143`) guard with only `decoded.includes("..")` then `join(INTAKE_DIR, sub, decoded)` / `join(CASE_DIR, "extracted", "pages", rest)`. **No symlink resolution and no absolute-path rejection** — a symlink planted under `files/`, `pages/`, or `extracted/pages/` pointing outside the root would be followed and served. The robust `safeJoin` exists in the sibling module but is not reused here. Exploit precondition: attacker-controlled symlink inside the case dir + localhost reach → **Medium-Low** for this threat model; should reuse `safeJoin`.
- **Zip-slip on intake [PASS].** `reader.py:46–60` `extract_zip` pre-scans `zf.namelist()`, raising `ValueError` on any `is_suspicious_entry` *before* `extractall`. `models.py:80–84` rejects `pure.is_absolute()` or `".." in pure.parts` after `\`→`/` normalization. (Does NOT resolve in-zip symlink entries, but `extractall` does not create symlinks from zip metadata by default, so the `..`/absolute check is the relevant guard and it holds.)
- **Insecure default bind / no auth / no CORS [Medium, local-first → effectively Low].** `Bun.serve({ port: PORT, ... })` (`index.ts:57–58`) sets no `hostname` → Bun default binds `0.0.0.0`, LAN/VPN-reachable. No auth, no CORS/Origin check; serves legal PDFs + full transcriptions. Doc claims "local, single-user." Fix: `hostname: "127.0.0.1"`.
- **Dependency CVEs [Low — no automated audit].** Floors pinned low, no upper bounds: `anthropic>=0.69`, `duckdb>=1.1`, `jsonschema>=4.0`, `pillow>=10.0`, `pypdfium2>=4.30`, `python-dotenv>=1.0`, `python-slugify>=8.0.4`, `rich>=15.0.0` (`pyproject.toml`). `pillow`/`pypdfium2` parse untrusted court files. No `pip-audit`/Dependabot/CI (Agent 00 confirmed no `.github/workflows/`). No known-exploited CVE identified in this pass.
- **Env separation / rotation [informational].** Single secret, single env (`ANTHROPIC_API_KEY`); no dev/prod split needed for a local tool. No rotation mechanism, but rotation = edit `.env`; acceptable at this scale.

## Scoring

**Dimension:** Security / configuration
**Score:** 7
**Rationale:** For a local-first single-user tool the fundamentals are right — no hardcoded or logged secrets, gitignored `.env`, fully escaped `Bun.$` interpolation (no command injection), and correct zip-slip protection. It loses points for a genuine, easily-fixable inconsistency: the static `/files`,`/pages`,`/png` handlers confine with a string `..` check instead of the robust `safeJoin` that already exists (no symlink resolution), and the server binds `0.0.0.0` with no auth while serving sensitive legal PDFs. Neither is a remote-exploit slam-dunk given the threat model, so this is a solid 7, not lower.
