# Custom Focus — Provenance & Mirror-Immutability Integrity

**Date:** 2026-06-25
**Reviewer:** jwogrady

## Finding

lawnlord's core claim — *mirror the court's deterministic intake zip exactly as an immutable base; everything is additive and traceable to source* — is **largely true at the database level and genuinely deterministic, but the "self-verifying" half of the claim is unimplemented, and one provenance pillar (the manifest's per-file sha256 + source URL) is silently discarded on import.** I read every write path end to end. The strongest result is append-only enforcement: there is **zero `UPDATE` anywhere in `src/`** (grep-confirmed), and the only `DELETE`s are scoped drop-and-rebuild within each layer's *own* tables — `ingest._clear_case` deletes only mirror rows it is about to re-insert (`ingest.py:44-58`), `explode` deletes only `pages`/`documents` (`explode.py:49-50`), `regions` deletes only `page_regions WHERE origin='pdf_text'` (`regions.py:131`). **`page_text` is never DELETEd or UPDATEd at all** — it is append-only by construction, and the surrogate-id PK `pt_<sha16>` of `(page_id|source|model|rev)` (`transcribe.py:78-86`) means re-inserting an identical variation is a PK conflict, so rev-0 of any variation is database-immutable. `force=True` correctly *appends* `_next_rev` (`transcribe.py:582,667`), never overwrites. Determinism is real: all content-hash IDs (`doc_<sha16>` `ingest.py:157`, `pt_<sha16>`, `pr_<sha16>` hashing structure only — never coordinates, `regions.py:78-84`), fixed-DPI PNG renders (`explode.py:24,57`), and — critically — **every `created_at` is the caller-supplied `generated_at`, never wall-clock.** I traced it: `captured_at()` derives `generated_at` from the manifest's `capturedAt` with a fixed `1980-01-01` sentinel fallback (`reader.py:100-109`), and it is threaded into *every* stored row (`ingest.py:89,142,173`, `explode.py:76,92`, `transcribe.py:575,612,689`, `regions.py:194`). `hashing.now_iso()` exists but is **never called anywhere** (grep-confirmed) — wall-clock is structurally absent from stored data. **Re-import IS byte-identical** for a given zip. This is the project's best dimension and it holds.

Layer separation is solid and mostly structural. Canonical-vs-derived is *not* merely a string column: the `source` discriminator (`'pdf_text'` vs `'ai'`) participates in the PK hash and in `_next_rev`'s `IS NOT DISTINCT FROM` match, so a derived `ai` reading *cannot* collide-overwrite or impersonate the canonical `pdf_text` variation — they occupy different surrogate-id rows. `pdf_text` fidelity is hardcoded `1.0` (`transcribe.py:575`) and `escalate_case` explicitly excludes `source='pdf_text'` from re-transcription (`transcribe.py:639-640,654`). The export anchor logic honors ADR-0008 exactly: `_anchor_index` (`export.py:117-126`) **always** returns the `pdf_text` reading as the canonical anchor when present, and only falls back to the AI consensus *medoid* when there is no `pdf_text` row — so AI consensus can never override the court's own text layer; the anchor always scores `agreement=1.0` and the AI readings are diffed *against* it (`export.py:138-154`). The web viewer keeps this visually unmistakable: `THE RECORD · PDF text layer` with a distinct `.record`/`badge-record` class is rendered **only** when `v.source === "pdf_text"` (`app.ts:548-549`; `styles.css:496-497,560-570`), AI readings render as derived comparison cells. Mirror purity holds: the 7 mirror tables are written *only* in `ingest.py` (verified — `explode`/`transcribe`/`regions`/`export` touch only their own additive tables or read mirror rows), so no additive layer can write the mirror today.

The two material gaps are both **provenance fidelity, not mutation**. First — **the "self-verifying via per-file sha256" claim is false in code.** The intake manifest carries a `files[]` array with a `sha256`, a source `url`, `bytes`, and a per-file `capturedAt` for all 22 filed PDFs (confirmed against the live `intake/odyssey-250914566/manifest.json`). **Nothing reads it.** `extract_zip` (`reader.py:46-60`) only checks entries for path-traversal — it never opens the manifest. `captured_at()` (`reader.py:100-109`) reads only the *top-level* `capturedAt`. `ingest_case` computes its own *fresh* `sha256_file` of the PDF bytes to mint the image id (`ingest.py:156`) but **never compares it against the manifest's declared hash** — so a tampered or corrupted PDF whose bytes differ from what the court captured would be ingested without complaint, just under a different `doc_<sha16>` id. The stored `images.sha256_hash` and `pages.page_image_sha256` columns are **written-once and never read back / re-verified anywhere** (grep-confirmed: both appear only in their `CREATE TABLE` and their `INSERT`). For a tool whose entire pitch is "provenance you can trust," the zip is *trusted*, not *verified*. Second — **the source-URL provenance chain is broken end to end.** The manifest's per-file `url` (where each PDF came from in the court portal) is dropped on the floor; `images` has no source-url column, so a `page_text` row can trace `page→image→event→case`, but the case-level `source_url`/`last_refreshed` (ingested at `ingest.py:88`, from `data.json` identity) is the *only* provenance that survives — and `export_actual` (`export.py:172-179`) does **not** even select `source_url`/`last_refreshed`/`source_note` into the viewer payload, so the canonical provenance the schema preserves never reaches the UI. Per-file capture URL/time is never persisted at all.

Compounding both: Agent 02's schema-versioning finding is **confirmed and provenance-material.** `apply_schema` (`db.py:231-237`) runs `CREATE TABLE IF NOT EXISTS` and writes `SCHEMA_VERSION` only `if row[0] == 0` — it **never reads or compares** the stored `schema_meta.version`. A v9 DB opened by v11 code silently gains the new tables but keeps `version=9` recorded, and any changed column on a pre-existing table is silently ignored. The working tree literally contains `lawnlord.duckdb.v9.bak` and `lawnlord.duckdb.pre-fresh.bak` (Agent 06) — the operator hit exactly this regenerate-by-hand path with no tooling signal. A DB that *misreports its own schema version* is a provenance defect by definition, even if the data within is correct.

## Evidence

- **No UPDATE in `src/` at all; DELETEs are layer-scoped drop-and-rebuild only** (grep) — mirror: `ingest.py:48-58`; exploded: `explode.py:49-50`; regions: `regions.py:131`. `page_text` has **no DELETE/UPDATE whatsoever**. *(enforced-by-construction)*
- **Append-only `page_text` is a DB guard, not convention:** surrogate PK `pt_<sha16>` of `(page_id|source|model|rev)` (`transcribe.py:78-86`); re-insert of an identical variation → PK conflict (rev-0 immutable). `force` appends `_next_rev` (`transcribe.py:582,667-668`), escalate appends `rev+1` (`transcribe.py:667`). *(enforced-by-construction)*
- **Deterministic timestamps — wall-clock structurally absent from stored data:** `generated_at` ← manifest `capturedAt` w/ fixed `1980-01-01` fallback (`reader.py:43,100-109`), threaded into every insert (`ingest.py:89,142,173`; `explode.py:76,92`; `transcribe.py:575,612,689`; `regions.py:194`). `hashing.now_iso()` is **defined but never called** anywhere (grep). Re-import is byte-identical. *(enforced-by-construction)*
- **Content-hash IDs, structure-only:** `doc_<sha16>` (`ingest.py:157`), `pt_<sha16>` (`transcribe.py:83-86`), `pr_<sha16>` hashes `(anchor_kind|anchor_id|span_index)` — *never* the possibly-model-supplied coordinates (`regions.py:78-84`). *(enforced-by-construction)*
- **Canonical anchor never overridden by AI consensus (ADR-0008):** `_anchor_index` returns `pdf_text` index if any (`export.py:123-125`), else the AI medoid; anchor always `agreement=1.0`, AI diffed against it (`export.py:138-154`). `pdf_text` fidelity hardcoded `1.0` (`transcribe.py:575`); `escalate_case` excludes `source='pdf_text'` (`transcribe.py:639-640,654`). *(enforced-by-construction)*
- **`source` is a structural discriminator, not just a label:** participates in the PK hash and `_next_rev`'s `... AND source = ? AND model IS NOT DISTINCT FROM ?` (`transcribe.py:493-498`) — an `ai` row cannot occupy or overwrite the `pdf_text` row's identity. *(enforced-by-construction)*
- **Web keeps canonical visually distinct:** `THE RECORD · PDF text layer` / `.record` / `data-record="1"` / `badge-record` rendered **only** for `v.source === "pdf_text"` (`app.ts:548-549`); `styles.css:496-497,535,560-570` style the record apart from derived AI cells. `download.ts:126,131` separate `pdf_text` vs model readings for export. *(enforced-by-construction)*
- **Mirror purity — 7 mirror tables written only in `ingest.py`:** `explode`/`transcribe`/`regions`/`export` write only their own additive tables or read mirror rows. **But** every write command opens the same RW handle + `apply_schema` (`cli.py` import/explode/transcribe/regions paths); nothing structurally forbids a future module from `UPDATE images`. *(convention-only — past `ingest.py` it rests on writers being well-behaved, as Agent 02 noted)*
- **GAP 1 — "self-verifying per-file sha256" is unimplemented:** manifest `files[]` carries `sha256`/`url`/`bytes`/`capturedAt` for all 22 PDFs (live manifest confirmed). `extract_zip` checks only path-traversal (`reader.py:46-60`); `captured_at` reads only top-level `capturedAt` (`reader.py:100-109`); `ingest` computes a *fresh* `sha256_file` for the id (`ingest.py:156`) but **never compares to the manifest hash**. Stored `images.sha256_hash`/`pages.page_image_sha256` are **write-once, never re-verified** (grep: appear only in CREATE + INSERT). The zip is trusted, not verified. *(NOT enforced — the claim is false in code)*
- **GAP 2 — source-URL provenance dropped:** manifest per-file `url`/`capturedAt` never persisted (no column on `images`); `export_actual` (`export.py:172-179`) does not select `source_url`/`last_refreshed`/`source_note`, so even the case-level provenance that *is* ingested (`ingest.py:88`) never reaches the viewer. Full chain `page→image→event→case→source_url` exists in the schema but is incomplete in practice and invisible in exports. *(convention-only / incomplete)*
- **GAP 3 — schema version never validated (confirmed, Agent 02):** `apply_schema` writes version only `if row[0] == 0`, never reads/compares (`db.py:231-237`); no migration path. Stale DB silently misreports its `schema_meta.version`. Live evidence: `lawnlord.duckdb.v9.bak`, `lawnlord.duckdb.pre-fresh.bak` in the tree. *(NOT enforced — silent misreport)*

## Scoring

**Dimension:** Provenance & invariant integrity
**Score:** 7
**Rationale:** The immutability and determinism invariants are genuinely strong and largely enforced by construction — no UPDATEs, append-only `page_text`, content-hash IDs, manifest-derived (never wall-clock) timestamps, and a canonical-anchor rule the export and viewer both honor so AI can never overwrite the court's own text. It is held back from 9 by the fact that the *self-verifying* half of the north-star claim is simply not implemented (the manifest sha256/URL are read by nothing; the stored hashes are never re-checked), the source-URL provenance chain is broken before it reaches the viewer, and the schema-version no-op lets a stale DB misreport its own provenance — three real gaps between the promise ("provenance you can *trust*/*verify*") and the enforcement ("provenance the writers chose to preserve").

## Notes to Next Agent

- **Most material violations for Synthesis (08), ranked:** (1) **"self-verifying per-file sha256" is unimplemented** — manifest hashes/URLs read by *nothing*, stored image/page sha256 *never re-verified* (`reader.py:46-109`, `ingest.py:156`). This is the single biggest gap between the stated value proposition and the code; pairs directly with Agent 04's "no round-trip test against the real zip." (2) **source-URL provenance is dropped** — per-file `url` never persisted, case-level `source_url` never exported to the viewer (`export.py:172-179`). (3) **schema-version no-op** lets a DB misreport its provenance (`db.py:231-237`) — confirmed across Agents 02/04/06/07; a stale-DB detector + a verify-on-import step are the two highest-value provenance fixes.
- **The good news to carry forward:** immutability/determinism/layer-separation are real and mostly structural — this is the project's strongest dimension and should be credited as such. The one convention-only weak spot (mirror writability past `ingest.py`) is latent, not yet violated.
**Critical:** No build/ship blocker. The provenance *integrity blocker for the claim itself*: lawnlord verifies the zip's structure (path-traversal) and its data contract (jsonschema) but **never verifies the zip's own per-file checksums and never re-checks the hashes it stores** — so "an immutable base you can trust" currently means "a base assumed trustworthy," not "a base proven intact." Implementing manifest-hash verification on import (and a schema-version gate) closes the gap between the north star and the engine.
